# 新人第一天

10 分钟内完成从零到能参与协作。

## 第 1 步：注册 GitHub

- 打开 <https://github.com/join>
- 注册并验证邮箱
- 把 GitHub 用户名发给 Lexa

## 第 2 步：接受邀请

- 接受对应仓库邀请
- 你可能只需要其中一个，也可能需要多个

## 第 3 步：读三页

按顺序看：

1. [仓库分工](./repo-scope)
2. [每周工作流](./workflow)
3. [GitHub 零基础入门](./github-beginner-guide)

## 第 4 步：准备一个 AI 入口

至少准备一个：

- ChatGPT
- Claude
- gstack office hours
- 其他你熟悉的外部咨询入口

默认规则不是“有问题先问人”，而是“先问 AI 把问题整理清楚”。

## 第 5 步：看一次真实上下文

去你要参与的目标仓库，看：

- `README`
- `TODO`
- open Issues

如果你看不懂 `TODO`，直接把内容贴给 AI，让 AI 用普通语言解释。

## 第 6 步：知道自己默认不需要什么

默认不需要：

- clone 仓库
- `git`
- VS Code
- Claude Code
- Codex CLI
- Node.js
- Vercel

## 第 7 步：你第一天就能做的三件事

- 看懂当前项目在推进什么
- 知道你的事情应该进哪个仓库
- 把一个模糊想法先整理成可提交的 Issue

## 完成标准

如果你已经能做到下面三件事，就算第一天完成：

1. 你能登录 GitHub
2. 你知道哪个仓库该提哪个问题
3. 你知道先问 AI，再开 Issue

## 出现这些情况怎么办

| 情况 | 立刻做什么 |
|------|------------|
| 我登不上 GitHub | 把截图发给 Lexa |
| 我不知道该去哪一个仓库 | 先看 [仓库分工](./repo-scope)，再问 AI |
| 我不知道这个需求值不值得提 | 先让 AI 帮你整理 |
| 我看不到 Issue 模板 | 截图发给 Lexa |
