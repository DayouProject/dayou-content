# 角色说明

这页不再假定固定岗位，而是只保留 Dayou 当前真正稳定的三种角色。

## 1. Collaborator / Requester

一句话：提出问题、观察机会、整理证据、验证结果的人。

负责：

- 发现问题或机会
- 先问 AI
- 把需求整理成 Issue
- 在结果出来后做验收

默认不负责：

- 写代码
- 部署
- 管环境变量
- 直接改生产配置

## 2. Lexa

一句话：技术实现者和最终收口人。

负责：

- 技术判断
- 排优先级
- 实现代码
- 上线
- 对齐跨仓库的一致性

默认不负责：

- 替所有模糊想法补完上下文
- 替每个人做前置调研

## 3. AI / 外部咨询

一句话：把模糊问题先变成清楚问题的前置层。

负责：

- 帮你定义问题
- 帮你补充可能的实现路径
- 帮你压缩成更好的 Issue 草稿

默认不负责：

- 替代正式留痕
- 替代最终产品判断
- 替代上线与实现

## 一句话边界

```text
Collaborator 提问题并验证结果。
AI 帮忙把问题问清楚。
Lexa 负责技术实现和上线。
```
