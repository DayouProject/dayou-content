# 角色说明

这份页面只回答一件事：3 个人分别交付什么，边界在哪里，什么叫做合格。

## Lexa（AlyciaBHZ）

一句话：负责把“知识库输入”变成“可发布资产”，拥有整条技术与生产链路。

| 项目 | 说明 |
|---|---|
| 每周交付物 | 1. 接收并确认 1 个本周内容输入包，内含 2 到 3 个概念。 2. 产出首批 3 到 7 条脚本。 3. 生成至少 3 条可发布视频。 4. 清掉本周所有技术阻塞。 |
| 使用工具 | `pipeline.py`、GitHub Issues、`dayou.dev`、Vercel、HeyGen、n8n |
| 明确不做 | 不长期写内容概念。 不负责 TikTok/Instagram/Reddit 日常运营。 不替知识库负责人做文化准确性判断。 |
| 每周投入 | 6 到 10 小时。重点在周一确认输入、周二到周四产出。 |
| 交接给谁 | 交给宣发负责人。交接时要在对应 Issue 留下脚本路径、视频路径、标题/说明建议、计划发布时间、是否还有风险。 |
| 做得好是什么样 | 脚本和视频按周推进；任何阻塞 24 小时内有明确处理人；没有“大家以为别人会做”的空档。 |

## 知识库负责人（TBD）

一句话：负责提供内容源头和文化把关，确保脚本有根、有边界、不过度翻译。

| 项目 | 说明 |
|---|---|
| 每周交付物 | 1. 每周一前提交 1 个 Content Idea Issue。 2. 每个 Issue 提供 2 到 3 个概念。 3. 每周审完首批 3 到 7 条脚本。 4. 每周至少更新 1 处主知识库备注或翻译规则。 |
| 使用工具 | GitHub 网页、Issue 模板、[CULTURAL_TRANSLATION.md](https://github.com/AlyciaBHZ/dayou-content/blob/main/CULTURAL_TRANSLATION.md)、微信 |
| 明确不做 | 不改代码。 不部署。 不运行 `pipeline.py`。 不接管 TikTok/Instagram/Reddit 账号。 |
| 每周投入 | 3 到 5 小时。重点在周一提交概念、周二到周三审核脚本。 |
| 交接给谁 | 先交给 Lexa。交接格式必须是 Issue，不是口头。每个概念至少写清经典来源、核心观点、现代角度、不要怎么讲。脚本审核时必须写“通过 / 需改 / 拒绝”和原因。 |
| 做得好是什么样 | 概念一看就能生成脚本；文化问题能具体指出句子和改法；发布后没有明显失真、误译或低级文化错误。 |

## 宣发负责人（TBD）

一句话：负责把成片送上平台、把反馈带回来，拥有发布执行和数据闭环。

| 项目 | 说明 |
|---|---|
| 每周交付物 | 1. 每周发布 3 到 7 条视频。 2. TikTok 与 Instagram 记录每条链接和基础数据。 3. Reddit 每天至少 1 条相关社区互动。 4. 每周提交 1 个 Weekly Report Issue。 |
| 使用工具 | TikTok 后台、Instagram 后台、Reddit、GitHub Issues、微信 |
| 明确不做 | 不改代码。 不部署。 不运行内容管线。 不替知识库负责人决定文化表述是否准确。 |
| 每周投入 | 5 到 7 小时。重点在周四接收资产、周五到周日发布和复盘。 |
| 交接给谁 | 把数据和用户反馈交回 Lexa 与知识库负责人。交接时要在 Weekly Report Issue 写清平台链接、播放、互动、异常、下周建议。 |
| 做得好是什么样 | 视频按计划发出；链接和数据可追踪；被限流、被举报、评论争议等问题当天可见；周报能直接反推下周题材。 |

## Git 协作规则

**只有 Lexa 能 push 到 main 分支。** 知识库负责人和宣发负责人都不需要 clone 仓库或装 git。

| 谁 | 能做什么 | 不能做什么 |
|----|---------|----------|
| 知识库负责人 | 在 GitHub 网页提 Issue、评论、写周报 | 不要 clone repo、不要 push 任何分支 |
| 宣发负责人 | 在 GitHub 网页看脚本、提 Issue、提周报 | 不要 clone repo、不要改配置文件 |
| Lexa | 全部 | — |

详见 [协作规则](https://github.com/AlyciaBHZ/dayou-content/blob/main/CONTRIBUTING.md)。

## 边界速查

```text
知识库负责人：负责内容源头与文化准确性，不碰代码与账号。
Lexa：负责系统、生产与策略，不接日常运营。
宣发负责人：负责发布、互动与数据，不碰代码与文化裁决。
```

## 交接标准

| 从谁交给谁 | 交什么 | 不能少什么 |
|---|---|---|
| 知识库负责人 -> Lexa | Content Idea Issue | 经典来源、2 到 3 个概念、现代角度、禁区提醒 |
| Lexa -> 知识库负责人 | 首批脚本 | 路径、版本、需要重点审核的句子 |
| Lexa -> 宣发负责人 | 成片与发布建议 | 视频路径、文案建议、发布时间、风险备注 |
| 宣发负责人 -> 全员 | Weekly Report Issue | 链接、数据、观众反馈、异常、下周建议 |
