# 角色说明

每个角色都只做自己最关键的部分。边界清楚，协作才不会混乱。

## 创始人 Founder

### 负责什么

- 拥有整体内容管线，包括脚本生成、视频生成、发布和部署。
- 制定内容方向、产品策略和协作规则。
- 判断流程是否顺畅，处理技术故障与发布异常。
- 决定优先级，确保本周内容按时上线。

### 使用什么工具

- GitHub 仓库与 Issues
- 本地内容管线
- HeyGen、n8n 等发布相关工具
- 微信，用于紧急沟通

### 不做什么

- 不长期代替内容负责人产出每周选题。
- 不替大师/顾问做传统文化准确性背书。
- 不负责日常逐条回复社媒评论或做运营录入工作。

### 每周时间投入

- 约 4 到 8 小时。
- 高峰期集中在周二到周四，因为需要跑管线、处理反馈、确认视频生成。

## 内容负责人 Content Lead

### 负责什么

- 每周提交内容灵感和选题方向。
- 审核脚本是否清楚、易懂、适合 TikTok/Instagram 等短视频平台。
- 跟踪每周内容表现，整理数据和下周建议。
- 管理社交账号的日常视角：看发布是否正常、看观众反馈、记录异常。

### 使用什么工具

- GitHub Issue，用来提选题、留反馈、写周报
- GitHub 网页，用来查看脚本与历史记录
- 微信，用于紧急同步
- 社交平台后台，用于查看播放、点赞、评论等表现

### 不做什么

- 不改代码、不部署系统、不修复自动化。
- 不直接改动管线配置或 API 设置。
- 不单独决定传统文化表述是否准确，相关判断需要大师/顾问参与。

### 每周时间投入

- 约 3 到 5 小时。
- 固定动作是周一提选题、周二到周三审核、周日写周报。

## 大师／顾问 Master Advisor

### 负责什么

- 每周提供 2 到 3 个适合传播的核心概念。
- 审核内容是否符合传统文化本义，避免误解、夸大或失真。
- 提醒哪些表达适合海外受众，哪些表达容易引起误读。

### 使用什么工具

- 微信，用于先口头或文字给出灵感
- GitHub Issue，用于留下可追踪的概念与修改意见
- GitHub 网页，用于查看脚本内容和历史讨论

### 不做什么

- 不负责发视频、不负责看账号数据。
- 不维护 GitHub Pages、内容管线或部署流程。
- 不承担日常项目管理或跨团队催办职责。

### 每周时间投入

- 约 1 到 3 小时。
- 重点在周一给方向，周三前完成文化准确性复核。

## 角色之间如何交接

- 内容负责人把选题整理成 GitHub Issue，交给创始人进入脚本生成。
- 创始人产出脚本后，先给内容负责人看传播表达，再交给大师/顾问看文化准确性。
- 大师/顾问给出修正意见后，内容负责人汇总成明确反馈，创始人据此定稿并推进视频生成。
- 发布完成后，内容负责人整理表现数据，创始人据此调整后续策略。

## 一句话判断边界

- 你负责“系统怎么跑”，那是创始人。
- 你负责“内容要不要过”，那是内容负责人。
- 你负责“文化表达准不准”，那是大师/顾问。
