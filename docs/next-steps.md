# 接下来 2 周

时间范围：2026-04-10 至 2026-04-24。

这页不再按固定岗位列任务，而是列 Dayou 这个系统当前最需要完成的几件事。

## 当前阶段目标

把 Dayou 从“靠口头和上下文推进”变成“靠统一的 Claude / Cursor / Codex 最低标准、清晰边界、Issue 留痕和 AI 辅助推进”。

## 这两周最重要的 5 件事

| 任务 | 目的 | 完成标准 |
|---|---|---|
| 建立跨仓库核心记忆 | 让 AI 不再漂移 | 三库边界、协作原则、优先顺序都有稳定记忆底稿 |
| 把 AI 配置页写清楚 | 让非技术 collaborator 知道从哪里开始 | Claude / Cursor / Codex 之一可被零基础用户直接上手 |
| 建立 AI 接管入口 | 让协作者把一个链接贴给 AI 就能继续推进 | AI 能接管 repo 路由、office hours、/daily、Issue 草稿 |
| 明确组织级 AI skills | 让不同 AI 客户端行为一致 | `/route`、`/issue`、`/office-hours`、`/daily` 已定义清楚 |
| 让非技术协作者默认走 AI → Issue 流程 | 提高需求质量 | 默认协作流程已写清并可执行 |

## 如果你是 collaborator，这两周你能帮上的事

- 看文档里还有哪些地方仍然像旧流程
- 测试一次 AI 接管入口是否真的能带你走完整流程
- 测试一次 `/daily` 和 `office hours` 的边界是否清楚
- 把你觉得模糊的地方提成 Issue
- 拿真实需求测试一次“先问 AI，再开 Issue”的流程
- 如果你发现一个需求很难判断属于哪个仓库，也提出来

## 本阶段完成线

如果下面 4 件事都成立，就说明这两周完成得不错：

1. 三个仓库边界已经清楚
2. 非技术 collaborator 知道怎么先配好 Claude / Cursor / Codex 之一
3. AI 接管入口已经能稳定带人进入 Dayou 流程
4. 非技术 collaborator 不再被默认要求装本地开发工具
5. 新需求默认能走 AI → GitHub Issue → Lexa 实现 这条线
