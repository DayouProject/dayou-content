# 接下来 2 周

时间范围：2026-04-10 至 2026-04-24。

这页不再按固定岗位列任务，而是列 Dayou 这个系统当前最需要完成的几件事。

## 当前阶段目标

把 Dayou 从“靠口头和上下文推进”变成“靠清晰边界、Issue 留痕和 AI 辅助推进”。

## 这两周最重要的 5 件事

| 任务 | 目的 | 完成标准 |
|---|---|---|
| 建立跨仓库核心记忆 | 让 AI 不再漂移 | 三库边界、协作原则、优先顺序都有稳定记忆底稿 |
| 统一 GitHub Pages 口径 | 让所有 collaborator 看到同一套说法 | 站点不再混用旧的固定角色叙事 |
| 明确三个仓库的 scope | 减少需求提错仓库 | `repo-scope` 页面可直接指导归属判断 |
| 让非技术协作者默认走 AI → Issue 流程 | 提高需求质量 | 默认协作流程已写清并可执行 |
| 把旧组织链接换成当前组织 | 降低入口混乱 | 主要链接切到 `DayouProject` |

## 如果你是 collaborator，这两周你能帮上的事

- 看文档里还有哪些地方仍然像旧流程
- 把你觉得模糊的地方提成 Issue
- 拿真实需求测试一次“先问 AI，再开 Issue”的流程
- 如果你发现一个需求很难判断属于哪个仓库，也提出来

## 本阶段完成线

如果下面 4 件事都成立，就说明这两周完成得不错：

1. 三个仓库边界已经清楚
2. GitHub Pages 站点口径统一
3. 非技术 collaborator 不再被默认要求装本地开发工具
4. 新需求默认能走 AI → GitHub Issue → Lexa 实现 这条线
