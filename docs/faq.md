# FAQ

这里优先回答非技术同学最常遇到的问题。

## 项目相关

### 这个项目到底在做什么？

大有内容团队把中国传统智慧转成适合海外短视频平台的内容。你不需要写代码，主要是提选题、审内容、看结果。

### 我需要懂命令行或 Git 吗？

不需要。大多数非技术角色只需要会用 GitHub 网页、微信和社交平台后台。

### 我应该先看哪一页？

先看 [角色说明](./roles)，再看 [每周工作流](./workflow)，最后根据需要看 [GitHub 零基础入门](./github-beginner-guide)。

## GitHub 相关

### 我注册好了 GitHub，下一步做什么？

把你的 GitHub 用户名发给项目负责人，等待仓库邀请。接受邀请后，就可以进入仓库看 Issues 和文档。

### 我不会写代码，也要用 GitHub 吗？

要，但你只用 GitHub 的网页部分：看文档、提交 Issue、评论反馈、写周报。

### Issue 模板没有显示怎么办？

- 先确认你打开的是仓库的 `Issues` 页面，而不是单个文件页面。
- 再确认点击的是 `New issue` 或 `新建 Issue`，不是普通评论框。
- 如果还是没有，直接微信联系创始人，并说明你看到的页面截图。

### 我写错了 Issue，可以改吗？

可以。打开你自己提交的 Issue，点击右上角 `...`，选择编辑即可。

## 流程相关

### 我应该把内容灵感发到哪里？

默认发 GitHub Issue。只有在临时、紧急、人在手机上不方便输入时，才先发微信，之后再补到 Issue。

### 脚本审核时，我到底看什么？

- 内容负责人看：是否好懂、是否适合短视频、是否有传播性。
- 大师/顾问看：是否符合传统文化本义、是否容易让海外观众误解。

### 什么时候该找创始人？

- 你无法登录 GitHub。
- 你看不到仓库或看不到 Issue 模板。
- 视频没按计划发布。
- 你发现流程堵住，但自己无法推进。

## 故障排查

### 我登不上 GitHub

- 先确认用户名或邮箱是否输对。
- 再试一次“忘记密码”重置。
- 检查邮箱里有没有 GitHub 的验证邮件或安全提醒。
- 如果仍然失败，把报错截图发给创始人。

### 我点开仓库提示没有权限

- 说明你可能还没接受邀请，先检查邮箱里的邀请信。
- 如果已经接受，退出 GitHub 后重新登录再试一次。
- 还不行，就把你的 GitHub 用户名发给创始人核对权限。

### 我找不到本周脚本

- 先看对应的 GitHub Issue 有没有说明本周脚本已生成。
- 如果还没生成，不是你的问题，说明流程还在创始人这一步。
- 如果确认已生成但你找不到路径，先在仓库里搜索 `workspace/scripts/`。

### 视频没有按计划发布

- 先在社交平台后台确认是否真的没发。
- 再去 GitHub Issue 或微信同步异常时间点。
- 这类问题由创始人处理；内容负责人负责记录影响和后续表现。

### 手机上看页面不方便怎么办？

GitHub Pages 站点和 GitHub 网页都可以用手机浏览器打开。若常用手机协作，建议安装 GitHub App，但仍以网页中的 Issue 模板为准。

## 还有问题怎么办

- 紧急问题：直接微信。
- 需要留痕和后续跟进的问题：发 GitHub Issue。
- 不确定问谁：先看 [角色说明](./roles) 判断归属。
