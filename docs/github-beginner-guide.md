# GitHub 零基础入门指南

> 从注册账号到正式参与 Dayou 协作。预计时间：15 分钟。

## 第一步：注册 GitHub 账号

1. 打开 <https://github.com/join>
2. 注册账号
3. 完成邮箱验证
4. 记住你的 GitHub 用户名

## 第二步：接受仓库邀请

Lexa 会邀请你进入需要的仓库。

你可能只需要其中一个，也可能需要多个：

- `mysterious`
- `animal-dayou`
- `dayou-content`

接受邀请后，你就能看代码、文档和 Issues。

## 第三步：先判断你该去哪一个仓库

不要一上来就在错误的仓库提需求。

先看 [仓库分工](./repo-scope)。

一句话判断法：

- 主产品：去 `mysterious`
- 宠物产品：去 `animal-dayou`
- 协作流程、文档、内容资产：去 `dayou-content`

## 第四步：熟悉 GitHub 页面

你主要会用这几个地方：

- `Code`：看文件和文档
- `Issues`：看需求、开需求、跟进需求
- `Pull requests`：大多数非技术 collaborator 不需要碰

## 第五步：先看上下文，再提需求

在开新 Issue 之前，先看：

1. 仓库的 `README`
2. 仓库的 `TODO`
3. 当前 open Issues

这一步的目的是避免重复提需求，也避免提一个已经在做的东西。

## 第六步：先问 AI，再开 Issue

如果你的想法还很模糊，先问 AI。

你至少要先问清楚：

1. 这个想法到底在解决什么问题
2. 大概率能不能做
3. 第一版最小可交付是什么
4. 什么结果算 done

然后再把整理后的版本放进 GitHub Issue。

## 第七步：在 Issue 里做什么

一个合格的 Issue 至少要有：

- 现状
- 想要的结果
- 为什么现在值得做
- 证据、链接、截图、例子
- AI 给你的整理结论
- done 标准

## 常见操作

### 怎么编辑我已经提交的 Issue？

打开 Issue，点击右上角 `...`，选择编辑。

### 怎么评论？

打开 Issue，在最下方评论区直接输入。

### 我能看到别人提的需求吗？

可以。打开仓库的 `Issues` 页面即可。

### 我会不会不小心改坏代码？

不会。正常浏览仓库文件是只读的。你主要写入的是 Issue 和评论。

## 手机上能不能用

可以。

- GitHub 网页可以用
- GitHub App 也可以用
- 正式提需求时，仍然优先用网页里的 Issue 模板

## 你默认不需要学什么

- 不需要学 `git` 命令
- 不需要学本地开发
- 不需要装 Claude Code
- 不需要装 Codex CLI
- 不需要碰 Vercel
